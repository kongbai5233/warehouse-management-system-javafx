# 仓储管理系统 | javafx-warehouse-management-system


## 项目概述

该系统是一套基于关系型数据库的仓储管理平台，用于管理仓库配件信息及操作记录。  
设计目标是通过数据结构建模、模块化功能划分和操作可追溯性，实现规范化的仓储管理，并提供可视化查询与审计功能。

---

## 核心设计理念

1. **数据驱动设计**  
   - 核心数据实体包括 **用户、仓库配件、操作记录**。  
   - 通过数据库表结构建模，将业务流程抽象为关系型数据，提高数据一致性和可操作性。

2. **模块化功能架构**  
   - 系统划分为登录/注册、配件管理、历史记录查询、用户管理和信息展示模块。  
   - 模块职责清晰，界面与业务逻辑解耦，支持后续功能扩展。

3. **操作可追溯性**  
   - 所有对仓库配件的增删改操作均写入历史操作记录表，通过关联用户表与配件表，实现操作链追踪。  
   - 查询模块可按操作人员和时间范围快速检索历史操作，实现审计与分析功能。

4. **技术选型与优化思路**  
   - **数据库**：MySQL，保证关系型数据完整性和事务一致性。  
   - **数据结构优化**：
     - 用户表和配件表采用主键约束保证唯一性  
     - 历史操作表建立复合索引（操作人员 + 时间）加速查询  
     - 外键约束保证操作记录与用户及配件数据一致性  
   - **系统逻辑**：前端输入校验与后端数据库验证结合，保证数据安全性  
   - **模块化设计**降低耦合度，便于未来增加权限控制或报表分析功能。

---

## 数据库结构

数据库结构及初始化数据已统一整理在 `data.sql` 文件中，主要表如下：

- **用户表**：存储账号信息及身份标识  
- **仓库配件表**：存储配件基本信息和库存状态  
- **历史操作记录表**：记录每一次增删改操作，包括操作人员、操作时间和操作类型  

> 数据库表设计支持快速检索和统计分析，同时保证数据完整性和操作可追溯性。

---

## 使用说明

1. 导入 `data.sql` 完成数据库初始化  
2. 系统支持账号注册、登录及修改用户名  
3. 可对仓库配件进行增删改查操作  
4. 历史操作查询可按操作人员检索  
5. 系统信息页面展示软件基本信息
